# What Employers Actually Test

In technical interviews, you will be asked to:

- Write code **without** Google or AI
- Explain **how** your code works
- Debug problems **live**
- Discuss **why** you chose an approach

**If you can't do it without help, you don't know it.**

---

# The Copy-Paste Trap

```
You → ChatGPT → Code Works → You Think You Learned

Reality:
ChatGPT learned → You copied → You learned nothing
```

Working code is **not** proof of learning.
Understanding is proof of learning.

---

# The Fundamentals Matter

These concepts will be used in **every single job:**

- How code flows (logic, loops, conditions)
- How data moves (variables, functions, APIs)
- How to read error messages
- How to break down problems
- How to debug systematically

**No AI can replace your thinking ability.**

---

# How to Actually Learn

## The Simple Rule

1. **Read** the concept
2. **Type** the code yourself (no copy-paste)
3. **Break** the code intentionally
4. **Fix** it without looking at the solution
5. **Explain** it to someone (or yourself)

If you can't explain it, you don't know it.

---

# Using AI the Right Way

## AI is a teacher, not a coder

**Good:**
- "Explain what this error means"
- "Why does this concept exist?"
- "What's the difference between X and Y?"

**Bad:**
- "Write the code for me"
- "Fix this without explaining"
- "Do my assignment"

# The BUILD Learning Framework

A simple 5-step cycle for each topic:

```
B - Break it down    (Read topic, understand the pieces)
U - Understand why   (Ask "why does this exist?")
I - Implement it     (Type code yourself, never copy-paste)
L - Learn from errors (Break it, debug it, fix it)
D - Demonstrate it   (Explain it out loud or write it down)
```

**Repeat this cycle for every concept in this course.**

---

# BUILD in Practice

| Step | What to Do | Time |
|------|-----------|------|
| **B**reak it down | Read the topic, identify key concepts | 15 min |
| **U**nderstand why | Research the problem it solves | 10 min |
| **I**mplement it | Write the code yourself | 30 min |
| **L**earn from errors | Intentionally break and fix | 15 min |
| **D**emonstrate it | Explain without looking at notes | 10 min |

**Total: ~80 minutes per concept**
